296 /*************************************************************************
297 KD-tree creation
298 
299 This subroutine creates KD-tree from set of X-values and optional Y-values
300 
301 INPUT PARAMETERS
302  XY - dataset, array[0..N-1,0..NX+NY-1].
303  one row corresponds to one point.
304  first NX columns contain X-values, next NY (NY may be zero)
305  columns may contain associated Y-values
306  N - number of points, N>=0.
307  NX - space dimension, NX>=1.
308  NY - number of optional Y-values, NY>=0.
309  NormType- norm type:
310  * 0 denotes infinity-norm
311  * 1 denotes 1-norm
312  * 2 denotes 2-norm (Euclidean norm)
313 
314 OUTPUT PARAMETERS
315  KDT - KD-tree
316 
317 
318 NOTES
319 
320 1. KD-tree creation have O(N*logN) complexity and O(N*(2*NX+NY)) memory
321  requirements.
322 2. Although KD-trees may be used with any combination of N and NX, they
323  are more efficient than brute-force search only when N >> 4^NX. So they
324  are most useful in low-dimensional tasks (NX=2, NX=3). NX=1 is another
325  inefficient case, because simple binary search (without additional
326  structures) is much more efficient in such tasks than KD-trees.
327 
328  -- ALGLIB --
329  Copyright 28.02.2010 by Bochkanov Sergey
330 *************************************************************************/
