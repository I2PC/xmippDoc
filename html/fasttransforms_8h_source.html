43 /*************************************************************************
44 1-dimensional complex FFT.
45 
46 Array size N may be arbitrary number (composite or prime). Composite N's
47 are handled with cache-oblivious variation of a Cooley-Tukey algorithm.
48 Small prime-factors are transformed using hard coded codelets (similar to
49 FFTW codelets, but without low-level optimization), large prime-factors
50 are handled with Bluestein's algorithm.
51 
52 Fastests transforms are for smooth N's (prime factors are 2, 3, 5 only),
53 most fast for powers of 2. When N have prime factors larger than these,
54 but orders of magnitude smaller than N, computations will be about 4 times
55 slower than for nearby highly composite N's. When N itself is prime, speed
56 will be 6 times lower.
57 
58 Algorithm has O(N*logN) complexity for any N (composite or prime).
59 
60 INPUT PARAMETERS
61  A - array[0..N-1] - complex function to be transformed
62  N - problem size
63 
64 OUTPUT PARAMETERS
65  A - DFT of a input array, array[0..N-1]
66  A_out[j] = SUM(A_in[k]*exp(-2*pi*sqrt(-1)*j*k/N), k = 0..N-1)
67 
68 
69  -- ALGLIB --
70  Copyright 29.05.2009 by Bochkanov Sergey
71 *************************************************************************/
72 void fftc1d(complex_1d_array &a, const ae_int_t n);
73 void fftc1d(complex_1d_array &a);

127 /*************************************************************************
128 1-dimensional real inverse FFT.
129 
130 Algorithm has O(N*logN) complexity for any N (composite or prime).
131 
132 INPUT PARAMETERS
133  F - array[0..floor(N/2)] - frequencies from forward real FFT
134  N - problem size
135 
136 OUTPUT PARAMETERS
137  A - inverse DFT of a input array, array[0..N-1]
138 
139 NOTE:
140  F[] should satisfy symmetry property F[k] = conj(F[N-k]), so just one
141 half of frequencies array is needed - elements from 0 to floor(N/2). F[0]
142 is ALWAYS real. If N is even F[floor(N/2)] is real too. If N is odd, then
143 F[floor(N/2)] has no special properties.
144 
145 Relying on properties noted above, FFTR1DInv subroutine uses only elements
146 from 0th to floor(N/2)-th. It ignores imaginary part of F[0], and in case
147 N is even it ignores imaginary part of F[floor(N/2)] too.
148 
149 When you call this function using full arguments list - "FFTR1DInv(F,N,A)"
150 - you can pass either either frequencies array with N elements or reduced
151 array with roughly N/2 elements - subroutine will successfully transform
152 both.
153 
154 If you call this function using reduced arguments list - "FFTR1DInv(F,A)"
155 - you must pass FULL array with N elements (although higher N/2 are still
156 not used) because array size is used to automatically determine FFT length
157 
158 
159  -- ALGLIB --
160  Copyright 01.06.2009 by Bochkanov Sergey
161 *************************************************************************/

165 /*************************************************************************

168 For given A/B returns conv(A,B) (non-circular). Subroutine can automatically
169 choose between three implementations: straightforward O(M*N) formula for
170 very small N (or M), overlap-add algorithm for cases where max(M,N) is
171 significantly larger than min(M,N), but O(M*N) algorithm is too slow, and
172 general FFT-based formula for cases where two previois algorithms are too
173 slow.
174 
175 Algorithm has max(M,N)*log(max(M,N)) complexity for any M/N.
176 
177 INPUT PARAMETERS
178  A - array[0..M-1] - complex function to be transformed
179  M - problem size
180  B - array[0..N-1] - complex function to be transformed
181  N - problem size
182 
183 OUTPUT PARAMETERS
184  R - convolution: A*B. array[0..N+M-2].
185 
186 NOTE:
187  It is assumed that A is zero at T<0, B is zero too. If one or both
188 functions have non-zero values at negative T's, you can still use this
189 subroutine - just shift its result correspondingly.

399 max(N,N)*log(max(N,N)) complexity is used (see ConvC1D() for more info
400 about performance).
401
